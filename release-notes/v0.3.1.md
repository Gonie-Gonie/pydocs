# docscriptor v0.3.1

## Highlights

- Added structured `AuthorLayout(...)` support so journal-style author blocks are the default for manuscript workflows while stacked profile layouts remain easy for guides and reports.
- Improved document rendering defaults:
  - DOCX now uses native page footnotes by default.
  - contents pages separate chapter-level entries more clearly.
  - captions stay visually closer to their tables and figures around page breaks.
- Reworked both bundled example documents:
  - the user guide is now a detailed reference-style document with cleaner cover matter and richer diagrams.
  - the journal paper now uses more meaningful figures and manuscript content tied to the actual workflow goals.

## Notes

- If you prefer the previous collected-notes behavior, use `Theme(footnote_placement="document")`.
- The release workflow now uses `release-notes/<tag>.md` automatically when that file exists, so tagged releases can publish curated release text without manual editing on GitHub.
